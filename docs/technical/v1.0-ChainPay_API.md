# ChainPay API 文档

> Go + Gin 后端 · Base URL: `http://localhost:8080/api/v1` · 2026年4月

---

## 概述

ChainPay 后端提供两类 API：
1. **员工档案管理**：雇主对员工数据的增删查
2. **发薪管理**：手动触发发薪、查询发薪记录

> **注意**：员工接收规则存储在链上合约，不通过后端 API 管理。

---

## 认证

当前版本（黑客松）使用**钱包地址**作为身份标识，通过请求头传递：

```
X-Wallet-Address: 0x1234...abcd
```

> 生产版本需引入签名验证（EIP-191 / Sign-In With Ethereum）

### 通用错误码

| 错误码 | 含义 |
|---|---|

## 员工档案 API

### 1. 添加员工

```
POST /api/v1/employees
```

**请求头**
```
X-Wallet-Address: 0x雇主钱包地址
Content-Type: application/json
```

**请求体**
```json
{
  "name": "张三",
  "nickname": "Alex",
  "wallet_address": "0x员工钱包地址",
  "salary_amount": "5000",
  "pay_frequency": "monthly",
  "cron_enabled": true
}
```

**字段说明**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| name | string | ✅ | 员工姓名 |
| nickname | string | ❌ | 昵称 |
| wallet_address | string | ✅ | 员工 EVM 钱包地址（0x 开头）|
| salary_amount | string | ✅ | 合约金额（USDC，字符串避免精度问题）|
| pay_frequency | string | ✅ | 发薪频率：`daily` / `weekly` / `monthly` |
| cron_enabled | bool | ❌ | 是否开启自动发薪（默认 true，全局开关也需开启）|

**成功响应** `201`
```json
{
  "code": 201,
  "message": "success",
  "data": {
    "id": 1,
    "employer_address": "0x雇主钱包地址",
    "name": "张三",
    "nickname": "Alex",
    "wallet_address": "0x员工钱包地址",
    "salary_amount": "5000",
    "pay_frequency": "monthly",
    "has_rules": false,
    "cron_enabled": true,
    "next_pay_date": "2026-05-01T00:00:00Z",
    "created_at": "2026-04-01T10:00:00Z"
  }
}
```

**失败响应**
```json
// 员工已存在
{ "code": 409, "message": "employee wallet address already registered", "data": null }

// 参数错误
{ "code": 400, "message": "invalid wallet address format", "data": null }
```

### 5. 验证员工身份（员工登录时调用）

```
GET /api/v1/employees/:wallet_address/verify
```

> 员工连接钱包后，前端调用此接口验证是否已被雇主注册

**成功响应** `200` — 员工存在
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "exists": true,
    "has_rules": false,
    "employer_address": "0x雇主钱包地址",
    "name": "张三",
    "salary_amount": "5000"
  }
}
```

**响应** `200` — 员工不存在
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "exists": false
  }
}
```

## 发薪 API

### 7. 手动触发发薪

```
POST /api/v1/payroll/execute
```

**请求头**
```
X-Wallet-Address: 0x雇主钱包地址
Content-Type: application/json
```

**请求体**
```json
{
  "employee_wallet": "0x员工钱包地址"
}
```

**字段说明**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| employee_wallet | string | ✅ | 员工钱包地址 |

> **注意**：前端不需要传 `lifi_call_data`。后端收到请求后，自动调用 Li.Fi REST API 为每条员工规则实时生成 callData，然后立即调用合约执行。这样时间差最小（< 1秒），避免 callData 因价格波动过期失效。

**成功响应** `200`
```json
{
  "code": 200,
  "message": "payout executed",
  "data": {
    "log_id": 42,
    "tx_hash": "0x交易哈希",
    "amount": "5000",
    "status": "pending",
    "created_at": "2026-04-01T10:00:00Z"
  }
}
```

**失败响应**
```json
// 员工未配置规则
{ "code": 400, "message": "employee has not set rules yet", "data": null }

// 合约执行失败
{ "code": 500, "message": "contract execution failed: insufficient allowance", "data": null }
```

## 定时任务说明

定时发薪由 Go 后端 `robfig/cron` 自动触发，**不通过 API 调用**。

```
触发逻辑：
  每天 00:00 UTC
    → 查询所有 next_pay_date <= now 的员工
    → 逐一检查 has_rules = true
    → 执行者钱包调用合约 executePayout
    → 写入 payroll_logs（trigger_type = "cron"）
    → 更新 next_pay_date（+1天 / +7天 / +30天）
```
